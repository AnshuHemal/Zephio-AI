What Zephio Needs Next — Prioritized by Impact
🔴 Critical (Revenue & Retention Blockers)
1. Stripe Payments

This is the single most important thing. You have a complete credits system, upgrade modal, pricing page, and plan tiers — but zero way to actually collect money. The upgrade button goes to /#pricing which does nothing. Every user who hits their 10-generation limit and wants to pay has no path forward. You're actively losing revenue right now.

What you need:

POST /api/stripe/checkout — creates a Stripe Checkout session and redirects
POST /api/stripe/webhook — handles checkout.session.completed, subscription.updated, subscription.deleted — updates user_credits.plan in the DB
POST /api/stripe/portal — opens the Stripe Customer Portal for billing management
A real upgrade button that hits the checkout endpoint
A "Manage billing" link in the dashboard header for Pro users
stripeCustomerId column on user_credits table to link users to Stripe customers
Add STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET, STRIPE_PRICE_PRO_MONTHLY to .env
Without this, everything else is polish on a product that can't make money.

2. Forgot Password Flow

Right now there's no "Forgot password?" link on the sign-in page. If a user forgets their password, they're locked out forever. This is a silent churn driver — users who can't get back in just leave. It's a 30-minute implementation using Insforge's built-in password reset email flow.

3. "Built with Zephio" Watermark on Free Exports

Every free export should have a small, tasteful badge in the footer — "Made with Zephio". Pro users get clean exports. This does two things simultaneously: creates a viral loop where every shared page advertises Zephio, and gives free users a concrete, tangible reason to upgrade. It costs nothing to implement and compounds over time. The export logic is already in 
export.ts
 — you just inject a small footer div into the HTML wrapper for free plan users.

🟡 High Impact (Retention & Engagement)
4. Project Folders / Organization

Right now all projects are in one flat list. A user who's been using Zephio for 3 months will have 30+ projects with no way to organize them. Adding simple folders (or tags) lets users group projects by client, type, or status. This is a retention feature — organized users feel in control and keep coming back. The dashboard already has search, so the infrastructure is there.

5. Duplicate Page Within a Project

Users can duplicate entire projects but can't duplicate a single page within a project. This is a very common workflow — "I like this page, let me make a variation of it." The page sidebar already has rename, delete, download, and history buttons. Adding a duplicate button there is a small addition with high daily-use value.

6. "Add New Page" Button in the Sidebar

Currently users can only add pages by prompting the AI. But sometimes a user wants to manually add a blank page or a page based on an existing one. A simple "+" button at the bottom of the page sidebar that either creates a blank page or prompts the AI to generate one would reduce friction significantly.

7. Project Last-Opened Sorting + "Continue where you left off"

The dashboard sorts by updatedAt which is good, but there's no visual "resume" affordance. When a user returns to the dashboard, they should see their most recent project highlighted or pinned at the top with a "Continue →" button. This reduces the cognitive load of "which project was I working on?" and improves time-to-canvas on return visits.

8. Prompt History / Suggestions

When a user is in the chat input, they have no memory of what they've typed before. Adding a simple dropdown that shows their last 5-10 prompts (stored in localStorage) lets them quickly reuse or iterate on previous prompts. This is especially useful for power users who have a set of prompts they use repeatedly. The chat input already has a popover for style presets — this fits naturally alongside it.

🟢 Polish & Delight (UX Quality)
9. Page Reorder Confirmation / Undo

Users can drag pages to reorder them in the sidebar, but if they accidentally drop a page in the wrong position, there's no undo for that specific action. The undo/redo system tracks page content changes but not order changes. Adding order changes to the history stack would make the reorder feature feel safe to use.

10. Canvas "Fit to Screen" / "Reset View" Button

The canvas has zoom in/out controls but no "fit all pages to screen" or "reset to default zoom" button. When a user has 5+ pages and has panned far away, there's no quick way to get back to a sensible view. A single button in the canvas controls that resets to the initial scale (26%) and position would be very useful. This is a one-line addition to the canvas controls.

11. Page Preview on Hover in Sidebar

The page sidebar shows page names and numbers but no visual preview. Adding a hover tooltip that shows a small thumbnail (using the same CSS-based ProjectThumbnail approach you just built) would let users quickly identify pages without clicking through them. This is especially useful for projects with 5+ pages where names like "Home", "About", "Contact" don't tell you much about the visual state.

12. Export to Figma / Copy CSS Variables

Power users — especially designers — want to take their generated designs into Figma or their own codebase. Adding a "Copy CSS variables" button that copies the rootStyles block to clipboard, and a "Copy HTML" button that copies the raw HTML, would make Zephio a better tool for professional workflows. These are zero-backend features that live entirely in the client.

13. Onboarding Checklist for New Users

New users land on the dashboard and see an empty state with example prompts. But there's no guided path that says "here's what to do next." A simple checklist — "Create your first project ✓", "Generate your first page ✓", "Export your design", "Share a preview link" — with completion tracking in localStorage would give new users a sense of progress and guide them toward the key value moments. Each completed step is a micro-win that builds habit.

14. "What's New" / Changelog Notification

When you ship new features (like keyboard shortcuts, version history, etc.), users who are already signed up have no way to know. A small notification dot on a "What's new" button in the dashboard header that shows a changelog modal would keep existing users engaged and aware of improvements. This is also a retention tool — users who know the product is actively improving are less likely to churn.

📊 Analytics & Business Intelligence
15. Admin Dashboard (Internal)

You have PostHog for user behavior, but no internal view of business metrics. A simple internal page (protected by an admin check) showing total users, active users this week, generations per day, free vs pro conversion rate, and top prompts would let you make data-driven decisions. This doesn't need to be fancy — even a simple /admin page with a few numbers would be valuable.

The One Thing
If you only do one thing from this list, do Stripe. Everything else is making a better product. Stripe is making a business. You have users, you have a pricing page, you have an upgrade modal — you're one integration away from revenue. Every day without it is money left on the table.
